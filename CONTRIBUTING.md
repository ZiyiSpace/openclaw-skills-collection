# Contributing to OpenClaw Skills Collection

感谢你对 OpenClaw 技能合集的关注！我们欢迎各种形式的贡献。

---

## 🤝 如何贡献

### 1. 报告问题

如果你发现了 bug 或者有改进建议，请在 GitHub 上提交 issue。

### 2. 提交新内容

我们欢迎以下类型的贡献：

- **技能测评**：分享你对某个 OpenClaw 技能的使用体验
- **新技能推荐**：推荐你认为有价值的技能
- **内容优化**：改进现有文案或结构
- **文档完善**：补充测试方法论、使用指南等

### 3. 分享数据

如果你有技能使用的数据（效率提升、准确率等），欢迎分享。

---

## 📝 内容规范

### 技能测评标准

所有技能测评应包含：

1. **基本信息**
   - 技能名称
   - 安装命令
   - 适用人群
   - 评分（⭐1-5）

2. **核心功能**
   - 主要特性
   - 使用场景
   - 优缺点分析

3. **实测数据**（真实测试）
   - 测试时长
   - 测试场景
   - 效果评估
   - 对比数据

4. **安装指南**
   - 安装命令
   - 验证方法
   - 常见问题

### 多平台适配规范

#### 小红书
- 避免竞品对比（拉踩）
- 数据保守但吸引眼球
- 添加免责声明
- 使用 emoji 和视觉化排版

#### 抖音
- 口语化表达
- 短句为主
- 突出数字和效果

#### 推特
- Thread 格式
- 英文为主
- 简洁明了
- 添加相关标签

#### GitHub
- 技术文档格式
- 详细测试方法论
- 数据来源说明
- Markdown 排版规范

---

## 🚀 提交流程

1. Fork 本仓库
2. 创建特性分支：`git checkout -b feature/your-feature`
3. 提交更改：`git commit -m 'Add some feature'`
4. 推送分支：`git push origin feature/your-feature`
5. 提交 Pull Request

---

## 📌 注意事项

- 所有数据必须真实可信，禁止虚构
- 避免平台规则禁止的内容（如小红书的拉踩测评）
- 保持内容的一致性和专业性
- 尊重开源协议和知识产权

---

有任何问题？欢迎提交 issue 或联系我们！
